## 5. Repository Structure

The repository is organized into the following top-level directories:

- `/logs` — laboratory execution records
- `/notes` — zettelkasten-style knowledge notes
- `/templates` — reusable document templates
- `/deliverables` — formal outputs (e.g. D1, D2)
- `/references` — bibliographic and external sources

## 6. Naming Convention

All files follow a timestamp-first naming convention to ensure chronological ordering in Git.

| Prefix | Document type    |
|--------|------------------|
| LOG    | Laboratory log   |
| ZN     | Zettel Note      |
| RPT    | Weekly report    |
| PROJ   | Project document |
| TPL    | Template         |
| D      | Deliverable      |
